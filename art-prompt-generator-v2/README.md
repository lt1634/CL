# DSE 視藝：練拆奪星王 · VA Star-Getter: Gen & Crack

動詞組合型品牌：「練」代表生成題目供練習，「拆」代表拆解扣題位。核心 Slogan：**「隨機生成模擬題，一鍵拆解扣題位。」** 針對香港 DSE 視藝科卷一乙部（創作），對齊四項評分準則，易於傳播（如：「今日你練拆咗未？」）。

---

## 功能

- **命題**：隨機模擬 DSE 風格有限制題目（如「成長」「衝突」「循環」「距離」「界限」等）
- **六塊輸出**：命題、Concept 諗法、視覺元素建議、構圖提示、技法建議、扣題引導
- **常見犯錯／注意**：部分題目附「犯錯／注意」提示，強調乙部須聯繫甲部評賞作品的物理特性或意涵，並運用構圖與光源策略提升作品質量
- **評分對照**：每塊標示對應 DSE 評分範疇（媒材及技法、視覺元素及設計原理、創意與想像力、主題傳意，各 20%）
- **簡潔模式**：只顯示命題 + 摘錄提示；頂部有句說明如何切回完整對照
- **圖解說明頁**：[docs/product-how-to.html](docs/product-how-to.html)（可列印／截圖，與商用 roadmap 的 [docs/infographic.html](docs/infographic.html) 分開）
- **收藏 / 複製 / 分享**：儲存心水題目、複製全文、分享
- **最近生成**：歷史記錄，可點擊還原
- **中英切換**：題目庫支援中文、英文
- **香港時間**：頁面顯示香港時區
- **PWA**：可安裝、離線使用（manifest + service worker）
- **Web Push（測試）**：頁底「Web 推播」區可訂閱並經伺服器發測試通知；另有「本機通知」不需後端

## PWA / Web Push 測試（Vercel）

1. **環境**：推播需 **HTTPS** 或 **localhost**。純 `file://` 開檔多數無法用 Push。
2. **安裝依賴**（部署前）：專案根目錄執行 `npm install`（使用 `web-push`）。
3. **環境變數**（見 `.env.example`）：
   - `VAPID_PUBLIC_KEY`、`VAPID_PRIVATE_KEY`：本機可跑 `npx web-push generate-vapid-keys` 產生
   - `VAPID_CONTACT_EMAIL`：例如 `mailto:你的信箱`
   - `PUSH_TEST_SECRET`：自訂一串密碼；頁面輸入框只會把這串傳到**你自己**嘅 `/api/push-send`，**勿**寫進 repo
4. **本機**：`npx vercel dev`（需在 `art-prompt-generator-v2` 目錄，並已登入 Vercel / 連到專案），再開瀏覽器測主頁。
5. **測試步驟**：允許通知 → 輸入 `PUSH_TEST_SECRET` → 「訂閱並發送測試推播」。成功會見系統通知。
6. **無後端時**：用「本機通知測試」或 Chrome **Application → Service Workers → Push**（自訂 payload）測試 SW 嘅 `push` 事件。
7. **iOS Safari**：PWA／推播支援度與桌面 Chrome 不同，建議主力用桌面 Chrome 做推播測試。

## 使用方法

1. 用瀏覽器打開 `index.html`，或部署到靜態空間（如 GitHub Pages、Vercel）
2. 按 **「✨ 生成題目」** 取得一條 DSE 風格題目：**同一分頁、同一語言**會先以「洗牌輪替」形式出題，題庫幾多條就一輪內各出一次，出晒先再重新洗牌（關閉分頁即重置）
3. 可切換 **簡潔模式** 只睇命題與方向提示
4. 需要時按 **★ 收藏**、**📋 複製** 或 **↗ 分享**

## 評分準則對照（四範疇各 20%）

| 評分範疇 | 對應輸出區塊 |
|----------|----------------|
| 媒材及技法的選擇和運用 | 技法建議 |
| 視覺元素及設計原理的選擇和運用 | 視覺元素建議、構圖提示 |
| 創意與想像力 | 命題、Concept 諗法 |
| 主題傳意 | 扣題引導 |

**畫面顯示：** 為便於掃讀，介面上將「視覺元素」與「構圖」**合併在同一色塊**（並以小標籤標示兩者）；**按「複製」時仍會輸出【視覺元素】、【構圖】兩行**，與題庫欄位一致。

主介面另有 **①〜④ 快速跳轉**、可摺疊「點解有①②③④？」說明，對齊 DSE 改卷語言。

所有題目均具備題庫中的各輸出欄位（對齊四項準則）；部分題目另附甲部聯繫、進階提示、色彩調性、常見犯錯／注意、難度。

## 題目來源與邏輯

- **歷年 DSE 試題** (2013–2025)：命題風格參考卷一 Part B 真題（感官與心理轉譯、時空與敘事性、身份與社會觀察）。部分題目經精煉以加強「DSE 味」（如《在裂縫中綻放的瞬間》《當冰川遇上熔岩：無聲的對峙》《指尖下的千里之外》《門縫間窺見的另一個世界》），並新增趨勢題（如《被人工智能取代後的畫室》《深海中的霓虹森林》）。
- **進階提示 (starTips)**：部分題目附 **★ 進階提示**，對應構圖透視與光源設計等應試要點。
- **甲部聯繫（評賞作品風格引導）**：部分題目附 **partALink**，融入 2014–2022 歷屆卷一甲部評賞作品風格（媒材與技法、構圖透視、光源設計、符號與意義），供乙部創作對應；例如畢加索、拉圖爾、霍克尼、金昌烈、馬格利特等。
- **色彩調性 (colourTone)**：部分題目附預設色彩建議（如冷色調、鄰近色、高飽和度），對應色彩構成。
- **常見犯錯／注意 (commonMistakes)**：部分題目附「犯錯」與「注意」列表，提醒考生避免常見失誤（如內外分界模糊、忽略甲部聯繫、光源過於平均等），並對應構圖與光源策略。
- **難度 (difficultyLevel)**：部分題目標示難度（中、中高、高），供考生選擇練習層級。
- 構圖提示區塊對應「視覺元素及設計原理」；進階提示區塊對應光源與構圖建議。

## 檔案結構

- `index.html` — 主頁與邏輯
- `prompts-dse.js` — 題目庫（32 題 × 中英，每題含命題／Concept／視覺／構圖／技法／扣題等；部分含 starTips）
- `docs/product-how-to.html` — **產品向**圖解（使用方法與準則對照）
- `docs/infographic.html` — 獲利化／路線圖資訊圖（與教學向分開）
- `manifest.webmanifest` — PWA 設定
- `service-worker.js` — 離線快取 + `push` / `notificationclick`
- `icon.svg` — 應用圖示
- `package.json` — 依賴（含 `web-push`，供 API 發推播）
- `api/vapid-public-key.js` — 公開 VAPID 公鑰（給瀏覽器訂閱）
- `api/push-send.js` — 以測試密鑰驗證後發送 Web Push
- `.env.example` — 環境變數範例
