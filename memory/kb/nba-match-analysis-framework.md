# NBA 單場連結 — 分析框架（對齊 Grok 式深度稿）

**用途**：當 `/poly` 或手動研究遇到 **NBA 單場／球隊對決** 連結（Polymarket 或其他公開頁）時，**每次**用同一套章節輸出，方便對照預測市場結算與「賽事本身」敘事。  
**非投注建議**；數字、傷兵、賠率以 **web_search／當日來源** 為準，過期要標註日期。

---

## 輸出章節（必跟順序）

### 1. 球隊當前戰績與形勢
- 兩隊 **戰績、分區排名、主客場**、**近 10 場**（或最近一段趨勢）
- **季後賽／附加賽動機**（邊隊有壓力、邊隊已無求）
- 一句 **敘事總結**（動機差異、心理／疲勞若明顯則寫）

### 2. 本季交手記錄
- **本季對賽次數**、日期、主客、比分、關鍵回合／球員（如有公開報導）
- **本场** 主客、若與歷史客場魔咒等有公開說法可寫，並標「參考敘事」

### 3. 傷病（最新）
- **分隊列出**：球員、位置、狀態（out / doubtful / day-to-day / 預計復出日期）
- 一句 **傷病對攻防影響**（邊隊缺口大）

### 4. 關鍵球員與數據對比
- 每隊 **2–4 名核心**：得分／板／助等 **賽季平均**（注明數據來源日期）
- **球隊級** 場均得分、籃板、助攻、FG%、三分%（若可取）
- 一句 **風格對比**（攻守、節奏、內線／外線）

### 5. 盤口、看點與預測（如有公開資訊）
- **讓分、總分 O/U、Moneyline**（若有；注明大致區間與來源類型，唔當聖旨）
- **看點** bullet：對位、動機、X factor
- **預測**：一句方向性 + **若比賽進行中／已完場** 要寫明狀態，唔裝作未發生

### 6. 總結
- 3–5 句：**邊隊敘事／實力／動機** 點影響 **預測市場**（若連結係 Polymarket，點返 **resolution** 風險：延期、規則字眼）

1. 先做本章節（**賽事本體**）。  
2. 再照 `polymarket-learning-playbook-research-prompt.txt` + Gamma：**市場 metadata、流動性、結算字句**。  
3. 最尾 **風險標籤** + **唔代替落單**。

## 學習要點（每次自我檢查）

| 檢查 | 原因 |
|------|------|
| 數字標日期 | NBA 每日變化大，舊數據會誤導 |
| 傷兵核對官方／隊記 | 賽前 1 小時仍可變 |
| 區分「模型機率」同「事實」 | 賠率係共識唔係賽果 |
| Polymarket 結算唔等於「邊隊贏」 | 睇清係 moneyline、spread、player prop 定其他 |
