# 設計原則庫（Design Principles）

用途：當 agent 提方案/改 UI 時，用呢度做「判斷依據」，避免只講「靚啲/modern」。

> 規則：每條原則盡量用「可檢查」語句（你睇得出做唔做到）。

---

## P0（高優先級）

### 1) 層級優先（Hierarchy first）

- 任何畫面必須清楚回答：**用戶第一眼要做/睇到乜**。
- 最少有 3 層層級：**標題** / **主要內容** / **次要信息**（用字級、粗細、顏色、留白分層）。

### 2) 留白係功能（Whitespace is structure）

- 用留白分區，而唔係靠一堆線條/卡片堆疊。
- 間距要「有系統」：同一層級用同一套 spacing（例如 8/12/16/24）。

### 3) 可訪問性底線（A11y baseline）

- 文字對比要夠（特別係灰字/按鈕 disabled）。
- 鍵盤焦點順序合理；focus 樣式唔可以消失。

---

## P1（中優先級）

### 4) 文案即介面（Copy is UI）

- 按鈕文案用動詞 + 結果（例如「保存更改」、「新增地址」），避免「確定/提交」。
- 錯誤提示要講 **用戶可做咩**，唔好只講「失敗」。

### 5) 預設值要幫人（Good defaults）

- 新手路徑要清晰；空狀態要提供下一步（CTA）。
- 表單預設值/placeholder 要減少認知負擔。

---

## 加新原則（你每次糾正就加一條）

格式建議：

- **原則名**：一句話
  - **原因**：點解重要
  - **做法**：點做到（可檢查）
  - **例子**：好/壞例（可選）
