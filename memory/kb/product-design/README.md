# Product Design（最小移植骨架）

呢個目錄係將「產品設計規範當上下文」嘅思路，**用最小成本**移植入你而家嘅 CL repo。

目標：

- 令 agent 做 **需求/原型/前端 UI** 時，有固定規範可跟（唔使次次重複教）
- 令「設計知識庫」可以慢慢自進化（每次復盤沉澱返嚟）

> 用法：當你要 agent 做產品設計/前端 UI，叫佢先讀 `memory/kb/product-design/` 再開始。

---

## 檔案一覽

- `01-design-principles.md`：設計原則庫（可重用判斷）
- `02-interaction-patterns.md`：交互模式庫（常用流程模板）
- `03-copy-guidelines.md`：文案規範（語氣、錯誤提示、按鈕文案）
- `04-glossary.md`：術語表（保持一致用字）
- `templates/`：原型/文檔模板（可選，按需要再加）

---

## 建議工作流（最小版）

1. **需求輸入**：你用一句到三句講清楚「目標用戶、場景、成功條件」。
2. **agent 先對齊規範**：引用上述 4 份文件（尤其原則/文案/術語）。
3. **產出**：先出資訊架構/flow + 介面草案，再落到具體 UI 文案/元件。
4. **復盤沉澱**：每次你糾正/發現更好做法，就加一條到相應文件。
