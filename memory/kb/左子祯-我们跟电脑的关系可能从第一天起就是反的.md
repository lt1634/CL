## 一句核心

> 傳統操作系統接收**指令**（instructions），AI 操作系統接收**目標**（objectives）。

我哋同電腦嘅關係可能從頭就反咗：係人去適應機器；將來應該係電腦懂你，你講目標就得。

---

## 要點

- **「翻譯」成本**：意圖一句，但要拆成十步操作；好多時間唔係思考，而係把想法翻譯成電腦能理解嘅步驟。按鈕、選單、彈窗本質係一套翻譯系統——因為電腦聽唔懂人話。
- **OpenClaw / Agent demo 嘅局限**：好多係「AI 模擬人用滑鼠鍵盤操作介面」——睇屏、搵按鈕、模擬點擊。即係「冇馬的馬車」：人冇咗，但保留人操作電腦嘅全套流程，AI 行緊人行嘅彎路。真正該問：若 AI 係主角，「電腦」仲應唔應該係而家呢個樣？
- **指令 vs 目標**：接收指令 = 你是操作者，要學佢語言；接收目標 = 你是老闆，講結果就得，點做係佢事。過去四十年係第一種；「從你懂電腦到電腦懂你」嘅翻轉遲早會發生。
- **App 會點變**：唔會消失，但會從「你面對的介面」變成「AI 調用的能力」。你唔會講「打開 Excel」，而係「幫我睇上月銷售點」。App 變後廚，AI 係服務員，你點菜。
- **新稀缺能力**：當瓶頸係「做咩」而唔係「點做」——**你能不能清晰知道自己要咩**，會變成最值錢。AI 會壓縮「用操作忙碌逃避思考」嘅空間。
- **點解唔係大公司先做**：巨頭被現有收入綁住，往舊系統塞 AI，唔敢重新設計系統本身（創新者窘境）。
- **Personal Computer 新意思**：以前 = 一台**屬於你**的電腦；而家多咗 = 一台**理解你**的電腦。

## 同 CL / OpenClaw 嘅對照

- daily-self-improvement、主腦 review SBA：你俾嘅係**目標**（「每晚找一個改進點」「主腦改下屬設定」），agent 自己揀點做、改邊個 .md → 接近「接收目標」。
- 若 OpenClaw 主要係 cron + 語意/記憶 + 改 .md，而唔係模擬點擊 GUI，就唔算「冇馬的馬車」式模擬人操作。
- 「知道自己要咩」同 Felix 學習分析裡「Markdown 裝住嘅 non-deterministic process」同向：值錢嘅係意圖同流程，唔係逐步操作。
